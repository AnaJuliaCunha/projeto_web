GitHub Pages

Sites Estáticos

Um site estático é aquele com conteúdo estável, onde cada usuário vê exatamente a mesma coisa em cada página individual.

Sites Dinâmicos

Já um site dinâmico é aquele em que o conteúdo é carregado de forma dinâmica, permitindo que ele mude de acordo com o usuário.

GitHub Pages é um serviço de hospedagem de site estático que usa arquivos HTML, CSS e JavaScript diretamente de um repositório no GitHub AE e, como opção, executa os arquivos por meio de um processo e publica um site.

Passo a Passo de como configurar seu gitHubPages

Passo 01 - Em Github, acesse o repositório do seu site. Se você usa o Github Free, é importante que seu repositório esteja como público. Vá em configurações (Settings). Na seção code e automation, clique em Pages. Na opção Source, abra o menu suspenso e selecione uma fonte de publicação, ou seja, a branch que desejar.

Passo 02 - De forma opcional, você pode selecionar uma pasta para sua fonte de publicação. Para isso, clique no segundo menu suspenso que aparece depois do passo 1 e selecione uma pasta da sua fonte de publicação. Com isso feito, é só Salvar.

Passo 03 -Aguarde enquanto o Github Pages conclui o processo de colocar seu site no ar. Ele vai lhe dar uma url com domínio github.io e ela estará pronta para ser acessada quando aparecer um sinal de OK e a cor verde ao redor da url. Depois disso, é só acessar sua página!

Build e Deploy

Build

é o termo usado para identificar uma versão compilada de um programa. Ou seja, quando as linhas de código escritas em linguagem de alto nível são traduzidas para linguagem de máquina, que um computador consegue entender.

Deploy

Literalmente, significa implantação. Quando estamos falando em desenvolvimento de aplicações web a palavra ganha o sentido de enviar atualizações e/ou mudanças de um ambiente para outro. Ele marca as etapas do processo de construção ou mudanças de um software, por exemplo.

CI e CD

CI (Continuous Integration) e CD (Continuous Deployment ou Continuous Delivery) são práticas essenciais na área de desenvolvimento de software que visam melhorar a qualidade, eficiência e velocidade do ciclo de desenvolvimento de software. Embora sejam frequentemente mencionadas juntas, elas têm objetivos ligeiramente diferentes:

Continuous Integration (Integração Contínua - CI):

O que é: CI é uma prática na qual os desenvolvedores integram seu código em um repositório compartilhado com frequência, geralmente várias vezes ao dia.

Como funciona: Quando o código é enviado para o repositório, uma série de testes automatizados é executada para verificar se o novo código não quebra funcionalidades existentes e se está de acordo com os padrões de qualidade do projeto.

Benefícios: A Integração Contínua ajuda a identificar e resolver problemas mais cedo no ciclo de desenvolvimento, melhorando a colaboração da equipe e a qualidade geral do código.

Continuous Delivery (Entrega Contínua - CD):

O que é: CD é uma extensão da CI que se concentra em automatizar e aprimorar a entrega de software a ambientes de teste ou de produção.

Como funciona: Após a integração bem-sucedida, o código passa por um processo automatizado de compilação, teste e implantação em ambientes de teste. Se todos os testes forem bem-sucedidos, o código é potencialmente pronto para ser implantado em produção.

Benefícios: A Entrega Contínua acelera o ciclo de lançamento de software, reduzindo o tempo e os erros associados à implantação manual, tornando mais fácil para as equipes de desenvolvimento e operações (DevOps) colaborarem e entregarem valor rapidamente aos usuários finais.
